# SMC 策略技术文档

## 概述

SMC (Smart Money Concepts) 策略基于机构交易行为分析，通过识别市场中的关键结构点、订单流和流动性区域来捕捉高概率交易机会。

## 核心概念


### 1. 摆动结构 (Swing Structure)

摆动结构是识别市场趋势的基础，通过检测价格突破关键高低点来判断趋势方向。

#### 摆动点识别
- **Swing High (SH)**: 局部最高点，左右两侧价格都较低
- **Swing Low (SL)**: 局部最低点，左右两侧价格都较高

#### BOS (Break of Structure)
- **看涨 BOS**: 价格突破前一个 Swing High，确认上升趋势
- **看跌 BOS**: 价格跌破前一个 Swing Low，确认下降趋势

#### CHoCH (Change of Character)
- **看涨 CHoCH**: 在下降趋势中，价格突破前一个 Swing High，可能趋势反转
- **看跌 CHoCH**: 在上升趋势中，价格跌破前一个 Swing Low，可能趋势反转

### 2. 内部结构 (Internal Structure)

内部结构使用更小的窗口（5根K线）来识别短期趋势变化，用于入场时机确认。

### 3. 订单块 (Order Block)

订单块是机构大量下单的区域，通常表现为：
- **看涨 OB**: 最后一根下跌K线之前的上涨K线区域
- **看跌 OB**: 最后一根上涨K线之前的下跌K线区域

订单块作为支撑/阻力位，价格回踩到这些区域时可能产生反弹。

### 4. 公允价值缺口 (Fair Value Gap, FVG)

FVG 是价格快速移动时留下的缺口，代表市场的不平衡状态。

- **看涨 FVG**: 上涨K线的最高价 < 下一根K线的最低价
- **看跌 FVG**: 下跌K线的最低价 > 下一根K线的最高价

价格通常会回补 FVG，这些区域是潜在的入场点。

### 5. 等高/等低 (EQH/EQL)

- **EQH (Equal Highs)**: 多个高点在同一水平
- **EQL (Equal Lows)**: 多个低点在同一水平

这些区域代表流动性集中，价格突破后可能加速。

### 6. ATR (Average True Range)

ATR 衡量市场波动率，用于：
- 动态止损设置
- 止盈目标计算
- 市场状态判断（高波动/低波动）

## 策略逻辑

### 趋势判断

```
趋势判断：BOS/CHoCH 状态机 → 确定多空方向
```

1. 使用摆动结构（50窗口）识别主要趋势
2. 使用内部结构（5窗口）确认短期趋势
3. 双周期共振：要求内部趋势与摆动趋势一致

### 入场触发

```
入场触发：趋势同向的 FVG（公允价值缺口）未被缓解 + 价格回踩到 FVG 区间
```

1. 等待 FVG 形成
2. 检查 FVG 是否与趋势方向一致
3. 价格回踩到 FVG 区域时入场

### 入场确认

```
入场确认：internal_trend == swing_trend 双周期共振（避免矛盾信号）
```

- 确保内部结构和摆动结构趋势一致
- 避免在趋势不明确时入场

## 开仓逻辑详解（`engine/smc.py` → `_check_trade_signal`）

开仓判断在每根 K 线（或实时 tick）推入引擎时执行，完整流程如下：

### 第一步：前置条件检查

| 条件 | 说明 |
|------|------|
| `ATR > 0` | ATR 暖机完成（至少 `ATR_PERIOD` 根 K 线） |
| `swing_trend != NEUTRAL` | 摆动趋势已确立（经历过 BOS 或 CHoCH） |
| `internal_trend == swing_trend`（可选） | 双周期共振；由 `INTERNAL_CONFIRM=True` 控制, `internal_trend` 仍为 NEUTRAL 时跳过该检查 |

任一条件不满足，本轮不开仓。

---

### 第二步：遍历活跃 FVG 池，寻找匹配缺口

对每个 FVG 进行以下过滤：

1. **方向一致**：`fvg.bias == swing_trend`
2. **未触发**：`fvg.triggered == False`（同一 FVG 不重复开仓）
3. **未过期**：`当前 K 线索引 - fvg.created_bar <= FVG_MAX_AGE`（默认 50 根，超过则淘汰）

---

### 第三步：判断价格是否已回踩至 FVG 触发区

找到方向匹配的 FVG 后，判断触发模式（按优先级）：

#### 模式 A：大 FVG 分拆建仓（优先）

**触发条件**：`FVG_SPLIT_ENABLED=True` 且 `FVG 高度 >= ATR × 1.5`

价格进入 **FVG 近端区间**即触发：

| 方向 | 近端区间（收盘 K 线） | 近端区间（实时 tick） |
|------|----------------------|----------------------|
| 看涨 | `fvg_mid < close <= fvg.top` | `fvg_mid < 当前价 <= fvg.top` |
| 看跌 | `fvg.bottom <= close < fvg_mid` 或 `high >= fvg.bottom 且 close <= fvg_mid` | `fvg.bottom <= 当前价 < fvg_mid` |

触发后执行**分拆建仓**：
- **30%** 仓位：**市价单**立即开仓（价格已在 FVG 近端）
- **70%** 仓位：在 **FVG 远端内侧（中点 ± 20% × 高度）**挂**限价单**，等待更深回踩
  - 看涨: `limit = fvg_mid - 0.20 × fvg_size`（向底部偏移 20%）
  - 看跌: `limit = fvg_mid + 0.20 × fvg_size`（向顶部偏移 20%）
- 止盈止损统一按完整仓位设置

**限价单取消条件**（以下任一触发）：
1. 价格实体收盘离开 FVG 区间（`close > fvg.top` 或 `close < fvg.bottom`）
2. FVG 达到 `FVG_MAX_AGE` 过期
3. 同向 BOS/CHoCH 已确认（趋势已开启，无需继续等待回踩）

#### 模式 B：标准 FVG 中点回踩（常规）

价格到达 **FVG 中点或更深区间**：

| 方向 | 触发区间（收盘 K 线，含插针捕获） | 触发区间（实时 tick） |
|------|----------------------------------|----------------------|
| 看涨 | `fvg.bottom <= close <= fvg_mid` 或 `low <= fvg_mid 且 close >= fvg.bottom` | `fvg.bottom <= 当前价 <= fvg_mid` |
| 看跌 | `fvg_mid <= close <= fvg.top` 或 `high >= fvg_mid 且 close <= fvg.top` | `fvg_mid <= 当前价 <= fvg.top` |

> 收盘 K 线模式额外捕获**插针入场**（K 线最低/高价扫过中点但收盘已出区间的情况）。

触发后以 **FVG 中点**作为入场价执行**市价单**开仓。

---

### 第四步：止损计算

`entry_price = (fvg.top + fvg.bottom) / 2`

#### 做多止损（BULLISH）

```
结构止损 = swing_low - ATR × 0.5
OB支撑止损 = 最近看涨OB下沿 - ATR × 0.3

最终止损 = min(结构止损, OB支撑止损中更紧的)
硬限 = min(最终止损, fvg.bottom - ATR × 0.1)   # 确保在 FVG 下方
```

OB 止损选择规则：`OB止损 > fvg.bottom - ATR×0.1` **且** `OB风险 < 结构风险` → 使用 OB 止损（更紧），否则用结构止损。

#### 做空止损（BEARISH）

```
结构止损 = swing_high + ATR × 0.5
OB阻力止损 = 最近看涨OB上沿（作为阻力） + ATR × 0.3

最终止损 = max(结构止损, OB阻力止损中更紧的)
硬限 = max(最终止损, fvg.top + ATR × 0.1)   # 确保在 FVG 上方
```

---

### 第五步：止盈计算（自适应）

**止盈倍数自适应**（`TP_ADAPTIVE=True`）：

```
ratio = ATR_当前 / ATR_前一根
ratio < 0.8  → tp_mult × 0.6  （低波动, 缩短目标）
ratio > 1.3  → tp_mult × 1.5  （高波动, 放大目标）
否则          → tp_mult × 1.0  （正常）
```

#### 做多止盈

```
ATR止盈 = entry + ATR × tp_mult(自适应)
OB阻力止盈 = 最近看跌OB上沿 - ATR × 0.3

优先 OB 止盈（当 OB止盈 > entry 且 R:R_OB >= R:R_ATR 时）
否则使用 ATR 止盈
```

#### 做空止盈

```
ATR止盈 = entry - ATR × tp_mult(自适应)
OB支撑止盈 = 最近看涨OB下沿 + ATR × 0.3

优先 OB 止盈（当 OB止盈 < entry 且 R:R_OB >= R:R_ATR 时）
否则使用 ATR 止盈
```

---

### 第六步：盈亏比过滤 & 信号发出

```
risk   = |entry - stop_loss|
reward = |take_profit - entry|
R:R    = reward / risk

R:R < 1.0          → 丢弃信号（不论市场环境）
R:R 1.0 ~ 1.5      → 仅当 MTF 高级别趋势对齐时允许开仓
R:R >= 1.5         → 正常开仓
```

通过所有过滤后：
1. 生成 `TradeSignal`（含方向、入场价、止损、止盈、ATR、触发 FVG、趋势依据）
2. 将触发 FVG 标记为 `triggered=True`，防止重复信号
3. 信号传递给执行层下单

---

### 开仓流程总览

```
每根K线/tick
    │
    ├─ ATR = 0 或 swing_trend = NEUTRAL ？ → 不开仓
    │
    ├─ INTERNAL_CONFIRM 且 internal ≠ swing ？ → 不开仓
    │
    └─ 遍历 FVG 池
           │
           ├─ 方向不符 / 已触发 / 已过期？ → 跳过
           │
           ├─ 大FVG + 价格在近端？ → [分拆建仓] 30%市价 + 70%限价@远端内侧(中点±20%高度)
           │
           └─ 价格在中点附近？ → [标准建仓] 市价@中点
                   │
                   ├─ 计算止损（结构/OB，取更紧，硬限FVG边界外）
                   ├─ 计算止盈（ATR自适应/OB，取R:R更优）
                   ├─ R:R < 1.0？ → 丢弃（强制）
                   ├─ R:R 1.0~1.5 且 MTF 未对齐？ → 丢弃
                   └─ 发出 TradeSignal，FVG.triggered = True
```

---

### 止损设置

```
止损：结构止损（Swing HL/HH ± ATR×0.5），OB 支撑阻力位更紧时优先使用
```

1. 优先使用订单块作为止损位（更紧）
2. 使用摆动高低点 ± ATR×0.5 作为结构止损
3. 动态调整，适应市场波动

### 止盈设置

```
止盈：OB 阻力/支撑位（R:R 更优时）或 ATR × tp_mult（自适应缩放：低波动 ×0.6，高波动 ×1.5）
```

1. 优先使用订单块作为止盈位（盈亏比更优）
2. 使用 ATR × tp_mult 作为默认止盈
3. 根据波动率自适应调整止盈倍数

### 盈亏比过滤

```
盈亏比过滤（分级）：
  R:R < 1.0      → 强制丢弃（任何市场环境均不开仓）
  R:R 1.0~1.5   → 仅当 MTF 高级别趋势明确对齐时允许开仓
  R:R >= 1.5    → 正常开仓
```

- 市价单手续费（入场+出场各一次）会进一步压缩实际期望值，因此 R:R < 1.0 无论如何都不值得交易
- R:R 1.0~1.5：仅当 MTF 高级别趋势明确对齐（`smc-enhanced` 专用）时允许
- 确保长期盈利性

## 风险管理

### BTCUSDT 维持保证金率层级

| 杠杆 | 仓位名义价值(USDT) | 维持保证金率 |
|------|-------------------|-------------|
| 150x | 0~300,000 | 0.004 |
| 100x | 300,000~800,000 | 0.005 |
| 75x | 800,000~3,000,000 | 0.0065 |
| 50x | 3,000,000~12,000,000 | 0.01 |
| 25x | 12,000,000~70,000,000 | 0.025 |
| 20x | 70,000,000~100,000,000 | 0.025 |
| 10x | 100,000,000~230,000,000 | 0.05 |
| 5x | 230,000,000~480,000,000 | 0.1 |
| 4x | 480,000,000~600,000,000 | 0.125 |
| 3x | 600,000,000~800,000,000 | 0.15 |
| 2x | 800,000,000~1,200,000,000 | 0.25 |
| 1x | 1,200,000,000~1,800,000,000 | 0.5 |

### 爆仓价计算

```
做多爆仓价 = 入场价 × (1 + 维持保证金率) - 有效余额 / 币种数量
做空爆仓价 = 入场价 × (1 - 维持保证金率) + 有效余额 / 币种数量
其中 有效余额 = 账户余额 - 入场手续费
```

### 熔断机制

- **单日亏损 ≥ 账户余额 n%** → 当日不再开仓（次日 UTC 自动重置）
- **连续亏损 n 笔** → 冻结 n 根 K 线不交易
- **盈利后连续亏损计数器归零**

## 增强型 SMC 策略

**smc-enhanced** 策略在基础 SMC 策略上添加了以下增强特性：

### 1. 成交量确认

- **成交量突破检测**: 当前成交量 > 均线 × 倍数
- **成交量趋势方向确认**: 确保成交量支持价格方向

### 2. 多时间框架 (MTF) 过滤

- **更高周期趋势分析**: 分析更高级别的趋势方向
- **趋势一致性要求**: 要求高低周期趋势一致

### 3. 市场环境检测

- **ADX 趋势强度分析**: 使用 ADX 指标判断趋势强度
- **波动率状态检测**: 识别市场处于扩张还是收缩状态

### 4. 自适应滑点模型

- **基于波动率的滑点计算**: 高波动时滑点更大
- **成交量调整**: 考虑成交量对滑点的影响
- **市场状态调整**: 根据市场环境动态调整

### 5. 信号质量评分系统

- **综合评估**: 综合多个因素评估信号质量（0-100）
- **质量过滤**: 只交易高质量信号
- **评分因素**: 趋势强度、成交量、波动率、多周期一致性等

## 使用示例

### 基础 SMC 策略

```bash

python main.py BTCUSDT -b -s smc

# 实盘交易
python main.py BTCUSDT -l -s smc
```

### 增强型 SMC 策略

```bash
# 使用增强型 SMC 策略回测
python main.py BTCUSDT -b -s smc-enhanced

```

```bash
# 调整止损止盈倍数

# 调整摆动结构窗口

```

## 策略对比

| 特性 | SMC | SMC-Enhanced |
|------|-----|--------------|
| 趋势识别 | BOS/CHoCH | BOS/CHoCH + MTF |
| 入场触发 | FVG + 双周期共振 | FVG + 双周期共振 + 成交量确认 |
| 止损止盈 | 结构止损 + ATR | 结构止损 + ATR + 自适应调整 |
| 信号过滤 | R:R < 1.0 丢弃；1.0~1.5 MTF 门控 | R:R 分级 + 质量评分 |
| 市场环境 | 无 | ADX + 波动率检测 |
| 滑点处理 | 固定 | 自适应滑点模型 |
| 适用场景 | 趋势市场 | 趋势市场 + 复杂环境 |

## 注意事项

1. **参数优化**: 不同交易对和时间周期需要调整参数
2. **市场环境**: SMC 策略在趋势市场中表现最佳
3. **风险控制**: 严格遵守止损和仓位管理规则
4. **回测验证**: 实盘前必须进行充分的历史回测
5. **监控维护**: 定期检查策略表现，及时调整参数
